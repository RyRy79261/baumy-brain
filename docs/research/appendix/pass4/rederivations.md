## Predictive Reconstructive Autobiographical Memory (PRAM), indexed on persons and the relat...
**Convergence with incumbent:** partial

**Frame:** Predictive Reconstructive Autobiographical Memory (PRAM), indexed on persons and the relationship. Re-derived from function-first memory science, memory is NOT a store queried for records but a continuously re-estimated predictive model of "what will I need to know or say next with THIS person." Its central theorem (Anderson & Schooler 1991, Reflections of the Environment in Memory, Psychol. Sci. 2:396-408): human memory has the shape it does because it is adapted to the statistics of retrieval demand, so the organizing quantity is need-probability, not storage. Its atomic unit is the person-indexed event-trace, carrying a fast-decaying verbatim surface, a durable gist, and its encoding context (who/when/where/mood/thread). Traces are captured one-shot and consolidated slowly into per-person semantic profiles; at recall the system RECONSTRUCTS (generates from gist + cue overlap + current goal) rather than reads a row — which is precisely what separates human-sounding recall from a robotic database lookup.

**Load-bearing structure:** A person/relationship-indexed autobiographical hierarchy (relationship-era -> recurring theme -> specific episode), after Conway & Pleydell-Pearce's Self-Memory System (2000, Psychol. Rev. 107:261-288), but with the "self" replaced by the RELATIONSHIP as the top-level organizing axis. Four flows hang on this spine: (1) a dual-rate pipeline (McClelland, McNaughton & O'Reilly 1995, Psychol. Rev. 102:419-457) - fast one-shot episodic capture that avoids catastrophic interference, feeding slow interleaved consolidation that distils context-bound episodes into context-free semantic profiles; (2) dual traces per event (Brainerd & Reyna fuzzy-trace theory) - verbatim decays fast, gist persists, so recall naturally becomes "you two argued about the dishes again, right?" not a quote; (3) cue-dependent reconstruction (Tulving & Thomson 1973, Psychol. Rev. 80:352-373, encoding specificity) - retrieval is context reinstatement/cue overlap, and the trace is re-written each time it is retrieved (Nader, Schafe & LeDoux 2000, Nature 406:722-726, reconsolidation); (4) a transactive meta-memory layer (Wegner, Giuliano & Hertel 1985/1991) - the system models who-knows-what and who-remembers-what across the people, and treats memory as a SHARED object co-constructed with the user, not a private world-model.

**Key commitments:**
- Need-probability is the master ranking signal, not similarity or co-occurrence weight. Activation/priority is an adaptive prior over future retrieval demand computed from recency, frequency, and spacing (Anderson & Schooler 1991). A memory surfaces because it is LIKELY to be needed in this conversation, not because it is lexically near the query.
- The atomic unit is the context-bound event-trace, not the fact/triple. Episodic-autobiographical material is primary; semantic profiles are a slow-consolidated DERIVATIVE of episodes, so the felt when/where/who-was-there/how-it-felt is preserved rather than flattened into edge metadata.
- Retrieval is reconstructive generation, not reproduction. Recall composes gist + partial cue + current goal into a plausible account (fuzzy-trace; Conway). This is the load-bearing source of naturalness AND of principled, human-like fallibility (approximate, self-consistent recollection instead of brittle exact-match).
- Dual-rate consolidation is mandatory to avoid the two failure modes: overwrite-everything (a companion with amnesia) and never-generalize (a companion that is just a transcript). Fast bind, slow integrate (CLS).
- Every retrieval is a write. Reconsolidation (Nader et al. 2000) means touching a memory in conversation makes it labile and updatable - the natural moment to revise, merge, correct, or re-weight it, not merely to strengthen a static edge.
- Memory is indexed by encoding context and reinstated by cue overlap (encoding specificity), so the index keys are participants, thread, time, place, and affect - not keywords. Naturalness = context reinstatement.
- The top-level organizing axis is persons-and-the-relationship, with a transactive meta-memory layer: the system tracks what each person knows/forgets/believes and treats the memory as a jointly held, co-authored record (Wegner). 'Marta always forgets bin day' is a first-class memory object.
- Gist survives, verbatim decays, and controlled forgetting is a feature: adaptive down-weighting of low-need traces IS the design, not data loss to be fought.

**What a conversational memory must do:**
- Rank candidate memories by predicted conversational need (recency x frequency x spacing x goal-relevance), so recall is proactive and situationally apt rather than a keyword hit.
- Store and prefer GIST while letting verbatim decay, so it can paraphrase shared history naturally ('the trip where it rained the whole time') instead of quoting stored rows.
- Reconstruct at recall under the current goal/topic, tolerating and gracefully signalling uncertainty ('I think it was Marta who...'), the way people actually remember.
- Capture in one shot from a single mention but consolidate slowly into durable per-person profiles, keeping fast recent detail and slow stable character in separate rates.
- Reinstate encoding context as the retrieval key: bring back who was there, the thread, the mood, the place - the cues that let a person remember, not just the topic string.
- Update memory ON retrieval - correct, merge duplicates, revise beliefs, and re-weight when a memory is surfaced in conversation (reconsolidation), so the record improves with use.
- Maintain a per-person, relationship-scoped model AND meta-memory (who knows/forgets/said what), and treat the store as a shared artifact the user can inspect, correct, and co-author.
- Forget adaptively - decay and prune low-need traces on purpose - so the working set stays relevant and recall stays fast and human-scaled.
- Preserve the felt/affective and temporal texture of episodes (this happened, it mattered, it felt like X), which context-free proposition stores discard.
- Distinguish 'remembering an episode' from 'knowing a fact' and use the right one: 'that time we...' vs 'you're vegetarian', because collapsing them makes the agent sound like a lookup table.

## Memory as a SELF-UPDATING GENERATIVE MODEL OF PERSONS-IN-CONTEXT — not a store of records....
**Convergence with incumbent:** partial

**Frame:** Memory as a SELF-UPDATING GENERATIVE MODEL OF PERSONS-IN-CONTEXT — not a store of records. Re-deriving from the molecular/systems science, the thing that keeps reappearing is that biological memory is not an archive that is written once and read back; it is a predictive, reconstructive model that (a) is organized around agents and relationships, (b) runs at two learning rates, (c) is gated by salience, and (d) is rewritten every time it is used. My organizing theorem: RETRIEVAL IS INFERENCE, NOT LOOKUP. The system's job at recall time is to CONSTRUCT a plausible, context-appropriate account from partial cues + accumulated schema, using the very same machinery it would use to imagine or predict — which is exactly why human recall feels alive rather than robotic (Schacter & Addis 2007, constructive episodic simulation; Bartlett's reconstructive tradition, formalized by Ghosh & Gilboa 2014). Everything else is subordinate machinery in service of that generative act. For a conversational companion, this reframes 'memory' from 'fetch the row about Marta' to 'reconstruct who Marta is to us, right now, given what we're talking about.'

**Load-bearing structure:** A four-layer coupled system, where the layers are defined by PROCESS and RATE, not by content type. (1) FAST EPISODIC INDEX — sparse, pattern-separated pointers that bind what/when/with-whom into a one-shot trace, storing an INDEX rather than the content itself (hippocampal indexing theory, Teyler & DiScenna 1986; Teyler & Rudy 2007). (2) SLOW SEMANTIC/SCHEMATIC SUBSTRATE — a dense, overlapping, distributed model that accumulates statistical structure gradually across episodes; this is where persons, relationships and gist live (the neocortical half of Complementary Learning Systems, McClelland, McNaughton & O'Reilly 1995). The two-rate split is load-bearing and NON-optional: fast one-shot learning and slow generalization are mathematically in tension and must be different subsystems. (3) CONSOLIDATION/REPLAY COUPLER — offline reactivation that interleaves episodes into the semantic substrate, abstracting specifics into schema and letting detail fade; crucially it is PRIORITIZED, replaying high-value experience preferentially (reward/novelty-prioritized sharp-wave-ripple replay, Ambrose/Pfeiffer & Foster line; schema-accelerated assimilation, Ghosh & Gilboa 2014). (4) SALIENCE GATE — a separate value/arousal channel that decides at encoding AND at replay what gets written and how strongly (amygdala/neuromodulatory modulation of consolidation, McGaugh 2004; behavioral tagging via synaptic tagging-and-capture, Frey & Morris 1997, letting a weak event be retroactively consolidated if it sits near a salient one). Wrapping all four: recall is CUE-DRIVEN PATTERN COMPLETION into RECONSTRUCTION, and every completion RE-LABILIZES and REWRITES the trace with present context (reconsolidation, Nader, Schafe & LeDoux 2000). A dedicated agent-identity substrate sits alongside the general episodic system (CA2 social-memory circuit, Hitti & Siegelbaum 2014; Meira et al. 2018) — persons are first-class, not just frequent nodes.

**Key commitments:**
- Retrieval is generative reconstruction, not fetch: recall synthesizes a context-appropriate account from partial cues + schema, using the same machinery as prediction/imagination (Schacter & Addis 2007). The system should be allowed to confabulate PLAUSIBLY and be graded on usefulness, with a separate provenance/confidence channel — the opposite of exact-record lookup.
- Two learning rates are mandatory: a fast, sparse, pattern-separated episodic index for one-shot capture, and a slow, dense, interleaving semantic substrate for generalization (CLS, McClelland et al. 1995). Collapsing them into one representation reintroduces catastrophic interference or kills one-shot recall.
- Store an INDEX/pointer, reconstruct the CONTENT: the fast layer holds sparse binding cues that reactivate distributed content, rather than holding the content verbatim (Teyler & DiScenna 1986). This is what makes recall recombinant instead of playback.
- Memory is write-on-recall: every retrieval re-opens the trace and rewrites it with current context (reconsolidation, Nader et al. 2000). Remembering is an update operation, not a read — shared history should drift and re-color with each conversation.
- Salience gates encoding and replay: an explicit value/arousal channel decides what is worth keeping and what to replay preferentially (McGaugh 2004; prioritized replay, Foster line). Uniform storage is biologically wrong and operationally wasteful; forgetting the unimportant is a feature.
- Consolidation is active abstraction into schemas, not archival: offline replay distills episodes into gist and person-models, and schema-consistent information assimilates fast while specifics fade on a gradient (Ghosh & Gilboa 2014).
- Cross-time capture: a temporally-proximal salient event can retroactively rescue a weak nearby memory within a window (behavioral tagging / STC, Frey & Morris 1997) — encoding strength depends on temporal neighborhood, not just the item.
- Persons and relationships are a first-class, dedicated substrate, not emergent from generic facts (CA2 social-memory circuitry). The model should be organized around agents and the observer's relationship to them.

**What a conversational memory must do:**
- Capture one-shot from a single mention (fast episodic index) without overwriting or destabilizing the accumulated model of a person — one-shot AND stable-generalization simultaneously.
- Reconstruct rather than recite: answer 'who is X to us' by synthesizing gist + a few vivid specifics into natural language, not by dumping stored rows — this is precisely the 'human recall, not database lookup' target.
- Bind every memory to who/when/with-whom context and let recall be cued by any partial fragment of that context (content-addressable pattern completion), including recalling a person from an oblique reference.
- Weight by emotional/relational salience: preferentially keep and resurface what mattered to the relationship (a promise, a conflict, a running joke) and let trivia decay gracefully.
- Update on use: when a memory is brought up in conversation and new context appears, revise the trace — the companion's account of shared history should evolve, not stay frozen at first write.
- Maintain a gist/detail gradient over time: recent episodes stay specific; older ones compress to schema ('Marta always X') while a few salient specifics survive — mirroring how humans narrate long relationships.
- Separate confidence/provenance from content, so constructive/plausible reconstruction can be surfaced with appropriate hedging rather than asserted as verbatim fact — the safety valve that makes generative recall trustworthy.
- Represent persons and relationships as first-class entities with their own model (identity, history, stance toward the observer), so recall is relationship-aware, not fact-aware.

## THE RELATIONAL CONSOLIDATION-GRADIENT FRAME: memory for a communicative companion is a two...
**Convergence with incumbent:** partial

**Frame:** THE RELATIONAL CONSOLIDATION-GRADIENT FRAME: memory for a communicative companion is a two-speed consolidation process organized around persons and shared history, read out by cue-driven RECONSTRUCTION rather than lookup. Concretely: (1) a fast, high-fidelity episodic trace layer captures conversational moments verbatim-ish with provenance; (2) an explicit offline consolidation process (the analog of sleep/hippocampal replay, per complementary learning systems — McClelland, McNaughton & O'Reilly 1995, Psychological Review) continually distils those traces into slow, gist-level knowledge about PEOPLE, RELATIONSHIPS, COMMITMENTS, and SHARED EPISODES — the ego-centric organization of Conway & Pleydell-Pearce's Self-Memory System (2000, Psychological Review), where autobiographical knowledge exists in service of the goals of an ongoing relationship, not as a general document corpus; (3) every trace carries a continuous, decaying ACTIVATION (recency x frequency x appraised salience) so availability tracks probability-of-future-need and forgetting is a tuned feature, not a failure (Anderson & Schooler 1991, Psychological Science — rational analysis of memory); (4) recall is Bartlett-style (1932) reconstruction: retrieval surfaces gist + provenance + a few vivid episodic anchors, and the agent RE-NARRATES them into talk ('oh, like that thing with your landlord last spring') instead of emitting fact rows — verbatim fidelity is reserved for the few items where stakes demand it (commitments, quotes, secrets); and (5) a COMMON-GROUND LEDGER tracks what has been mutually said and acknowledged (Clark & Brennan 1991 grounding; Hirst & Echterhoff 2012, Annual Review of Psychology, on how conversation itself reshapes memory; Wegner 1987 transactive memory for who-knows-what), because a companion's memory is one half of a two-person memory system — never re-explaining, never re-asking, able to allude and co-reminisce.

**Load-bearing structure:** The load-bearing element is NOT a taxonomy of stores — it is the pair of PROCESSES the stores fall out of: (A) the consolidation gradient (fast episodic capture -> slow offline distillation into person/relationship gist, with bitemporal supersession so 'you used to X' survives belief change) and (B) the activation economy (graded, decaying, salience-weighted availability governing every read and every forget). Plural stores (episodic traces, consolidated person/relationship knowledge, common-ground ledger, commitments/prospective queue) emerge as stations along the gradient, and the consolidated layer is naturally a weighted, provenance-carrying relational graph over persons — but the graph is an INDEX into evidence, not the memory itself; the reconstructive read step is what makes recall sound human rather than like a database hit. If you keep the stores but drop the two processes, you get exactly the robotic-lookup failure mode the objective forbids.

**Key commitments:**
- Two-speed learning: fast verbatim episodic capture plus a slow, explicit, offline consolidation pass that distils gist about people and relationships — consolidation is a scheduled first-class process, not a side effect of retrieval (complementary learning systems: McClelland, McNaughton & O'Reilly 1995).
- Person/relationship-centered indexing: the organizing axis is WHO (persons, dyads, shared episodes, commitments), not topics or documents — the machine analog of the Self-Memory System's working-self goals (Conway & Pleydell-Pearce 2000).
- Rational forgetting via a graded activation economy: availability = f(recency, frequency, appraised salience), tuned to probability-of-need; decay, gist-only recall, and occasional 'remind me?' are correct human-like behavior, not bugs (Anderson & Schooler 1991).
- Reconstructive readout: retrieval returns gist + provenance + episodic anchors as MATERIAL FOR GENERATION; the agent re-narrates rather than quotes; verbatim storage is the exception reserved for high-stakes items (Bartlett 1932; gist-outlives-verbatim per fuzzy-trace theory, Brainerd & Reyna).
- Bitemporal belief revision: facts about people change; supersession must preserve 'was true / believed when' so the agent can say 'you used to hate coffee' instead of silently overwriting or contradicting itself.
- Common-ground ledger as a distinct memory object: track what has been mutually said and acknowledged in BOTH directions (what I told them / what they told me), enabling allusion, co-reminiscence, and never re-explaining (Clark & Brennan 1991; Hirst & Echterhoff 2012).
- Transactive/provenance stance: every trace keeps source, speaker, and trust; the companion is one node in a two-person (or group) memory system, so who-said-what and who-knows-what are memory content, not metadata afterthoughts (Wegner 1987).
- Prospective, write-time appraisal: salience is assigned at encoding by asking 'what will future conversations need?' — memory exists to serve future communication, mirroring the constructive-episodic-simulation view that remembering and imagining-forward share machinery (Schacter & Addis 2007).

**What a conversational memory must do:**
- Model persons stably across months: identity, preferences, relationships, running life-threads, updated without fragmenting one person into several records.
- Recall shared episodes with temporal order and allude to them naturally in one clause ('like that landlord saga'), not as retrieved records.
- Maintain common ground: never re-explain what it already told this person, never re-ask what they already answered, and know which stories it has already told.
- Handle change gracefully: supersede beliefs bitemporally so it can reference the past state ('you used to...') without contradiction or silent amnesia.
- Forget like a person: graded confidence, gist surviving detail, low-salience material fading, and honest hedging ('I think you mentioned...') instead of confabulated precision.
- Preserve provenance and trust: who said it, when, firsthand vs forwarded — so misattribution and memory poisoning are structurally impossible, not just unlikely.
- Keep prospective memory: commitments, follow-ups, and open threads resurface at the right future moment unprompted.
- Protect the few verbatim-critical items (secrets, exact commitments, quotes) at full fidelity while everything else lives as gist.

## A two-speed, socially-indexed RECONSTRUCTIVE memory organized around the relationship: a f...
**Convergence with incumbent:** partial

**Frame:** A two-speed, socially-indexed RECONSTRUCTIVE memory organized around the relationship: a fast episodic experience stream plus a slowly consolidated layer of person models and shared-history narrative, threaded end-to-end by a common-ground ledger (who said/witnessed/acknowledged what), where retrieval is cue-driven reminding that reconstructs from gist with calibrated confidence, and forgetting is rational availability decay. The organizing theorem: memory for a companion is not a database of the world but the maintained state of a relationship — its purpose (per Bluck 2003's social function and Clark & Brennan 1991's grounding) is to keep common ground and shared history alive, so the load-bearing elements are the DYNAMICS (encoding appraisal, consolidation/semanticization, reconstruction, decay) and the SOCIAL INDEX, with stores as the substrate those dynamics require.

**Load-bearing structure:** The complementary-learning-systems split under a relationship-centric self-model: (1) a fast, provenance-rich, pattern-separated EPISODIC STREAM of interaction events (verbatim-leaning, rapid write, rational decay per Anderson & Schooler 1991); (2) a slow CONSOLIDATED LAYER — per-person models and a shared-relationship narrative — produced by an offline consolidation process that extracts gist across episodes while retaining a few vivid anchor episodes (McClelland/McNaughton/O'Reilly 1995; Tulving 1972 episodic-to-semantic); (3) a COMMON-GROUND LEDGER as first-class metadata on every trace: source, witnesses, mutual-acknowledgment status, audience/secrecy scope (Clark & Brennan 1991; Wegner 1987; Hirst & Echterhoff 2012). Retrieval sits on top as Schank-style reminding: conversational cues activate gist first, episodes as supporting evidence, and the reply is regenerated (Bartlett/Conway reconstruction), never quoted from storage. What is load-bearing is this triad of processes (appraise-and-encode, consolidate, reconstruct-with-decay) over the two stores plus the social index — remove any one and recall turns robotic, creepy, or confabulatory.

**Key commitments:**
- Two complementary stores at different timescales with an explicit consolidation process that semanticizes repeated episodes into stable person-facts while keeping a small set of vivid anchor episodes (Tulving 1972; McClelland, McNaughton & O'Reilly 1995)
- Primary organizing axis is persons and the shared relationship — not topics, not time; encoding and retrieval are modulated by the goals of the relationship, adapting Conway & Pleydell-Pearce's (2000) self-memory system to a dyadic 'relationship-memory system'
- Common-ground and provenance metadata on every trace (who said it, who was present, was it mutually acknowledged, audience scope) as a first-class index — required for audience design, privacy, and recall that reads as shared history rather than surveillance (Clark & Brennan 1991; Wegner 1987; Hirst & Echterhoff 2012)
- Reconstructive, confidence-calibrated retrieval: the store returns gist plus supporting episodes as material for regeneration, hedged in proportion to trace strength ('I think you mentioned...'); errors must be gist-preserving, never confabulated specifics (Bartlett 1932; Conway 2000; fuzzy-trace theory, Brainerd & Reyna — latter cited from knowledge, not re-verified)
- Rational forgetting as a feature: availability is a function of recency, frequency, and appraised salience/emotional weight, with graceful gist-level degradation before loss (Anderson & Schooler 1991)
- Reminding-style opportunistic retrieval via thematic/situational indices, enabling spontaneous 'that reminds me' recall rather than only query-answering (Schank 1982, Dynamic Memory)
- Appraisal-gated encoding: salience and relationship-relevance decide encoding depth; not everything is stored, and what is stored is stored unequally
- Evaluation by conversational function, not abstract recall: grounding efficiency, relationship continuity, naturalness and appropriateness of recall (Bluck 2003's self/social/directive functions, social primary)

**What a conversational memory must do:**
- Maintain and incrementally update common ground across sessions: know what is mutually known, who was present when it was established, and whether it was acknowledged
- Sustain durable, evolving models of each person and of the shared relationship (dispositions, preferences, ongoing concerns, our history together)
- Recall specific shared episodes as shared history ('that time we...') with correct source attribution, distinguishing 'you told me' from 'I inferred' from 'someone else said'
- Retrieve opportunistically from conversational cues (reminding), surfacing relevant history unprompted at the right moments, not only under explicit query
- Forget gracefully and hedge honestly: degrade to gist with calibrated uncertainty rather than either total recall (creepy/unnatural) or fabricated detail (trust-destroying)
- Consolidate: distil repeated episodes into stable semantic knowledge while preserving a few vivid anchors, so old material grounds identity-level statements ('you've always loved...') without replaying transcripts
- Respect social boundaries: track secrecy and audience scope so recall never leaks across people or contexts and never reads as surveillance
- Serve the conversation's functions — bonding, continuity of the relationship, directive usefulness — and be measured against those, not against QA-style recall benchmarks
