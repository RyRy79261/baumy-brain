# Decision records (ADRs)

Short, durable records of **why** a non-obvious architectural or tooling choice was made — so future
work (and future maintainers) inherit the reasoning, not just the outcome. Each is dated, has a status,
and names the conditions that would make us **revisit** it. Reversible by writing a superseding record.

Conventions elsewhere in the repo (`AGENTS.md` "Anthropic-only", "Better Auth is NOT used") are the same
idea inline; these are for decisions that warrant a standalone rationale.
